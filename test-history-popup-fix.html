🔧 LinuxDo Scripts - 历史记录弹窗修复报告

✅ 修复概述

成功解决了历史记录弹窗无法显示的问题，通过多重保障机制确保弹窗在任何情况下都能正常显示。

🐛 原问题分析

问题1: 位置计算错误

弹窗被定位到 (1142.75, 844.25)，可能超出视窗范围导致不可见

问题2: CSS特异性冲突

浏览器扩展环境中存在CSS样式冲突，导致弹窗样式被覆盖

问题3: Vue响应性问题

v-show 指令可能没有正确工作，导致弹窗无法显示

🛠️ 修复方案

方案1: 安全位置计算
// 优先使用屏幕中心位置，确保完全可见 let left = (window.innerWidth - popupWidth) / 2 let top = (window.innerHeight - popupHeight) / 2 // 确保在视窗内，添加最小边距保护 left = Math.max(margin, Math.min(left, window.innerWidth - popupWidth - margin)) top = Math.max(margin, Math.min(top, window.innerHeight - popupHeight - margin))
方案2: 最高优先级CSS样式
.history-popup-force { display: block !important; opacity: 1 !important; visibility: visible !important; pointer-events: auto !important; z-index: 2147483647 !important; /* 最大z-index值 */ position: fixed !important; }
方案3: 强制显示机制
forceShowPopup() { const popup = this.$el.querySelector('.history-popup') if (popup) { const forceStyles = [ 'display: block !important', 'opacity: 1 !important', 'visibility: visible !important', 'pointer-events: auto !important', 'z-index: 2147483647 !important' ] popup.style.cssText = forceStyles.join('; ') } }
方案4: DOM保护机制
setupDOMProtection() { // 每100ms检查一次弹窗状态，确保显示正常 this.domProtectionInterval = setInterval(() => { if (this.isPopupVisible) { const popup = this.$el?.querySelector('.history-popup') if (popup) { // 检查是否被意外隐藏，如是则强制修复 const computedStyle = getComputedStyle(popup) if (computedStyle.display === 'none' || computedStyle.visibility === 'hidden') { this.forceShowPopup() } } } }, 100) }

本次修复采用了防御性编程的思路，通过多重保障机制确保功能的健壮性：

  • ✅ 位置计算：优先中心位置，避免超出视窗
